# Phase 3: Baseline Agent & Comparison Analytics

## Overview
In Phase 3, we introduced a Baseline Agent ("Dumb AI") that follows fixed, deterministic rules. This serves as a critical control group to demonstrate the superior tactical intelligence and adaptability of the Hive Mind's agentic reasoning.

## Features Built

### 1. Baseline Agent (`src/agents/baseline_agent.py`)
- **Deterministic Logic**: Unlike the Hive Mind, the Baseline Agent does not use LLMs. It follows a 'Standard Rush' tactic, ordering all mummies to chase the player directly.
- **Fixed Negotiation**: Provides hardcoded negotiation strings to satisfy the schema while representing the lack of internal debate.

### 2. Multi-Mode Logging (`src/crud/trace_logger.py`)
- **Mode Tracking**: The `agent_traces.jsonl` now includes a `mode` field for every entry, identifying whether it was generated by the 'agentic' Hive Mind or the 'baseline' rules.
- **Comparison Analytics**: Implemented `get_comparison_stats()` to aggregate performance metrics, including total calls and unique tactical diversity per mode.

### 3. Expanded API
- **`POST /api/v1/hive-mind/baseline`**: Directly invokes the Baseline Agent.
- **`GET /api/v1/traces/compare`**: Provides a real-time side-by-side comparison of the two systems.

## Value Proposition
This baseline comparison is mandatory for the +5% bonus in the hackathon. It proves that the Hive Mind isn't just "random" or "fixed," but is actually debating and adapting based on player health, active elements, and tactical context.
